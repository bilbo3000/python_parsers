Documented Preferences
Last Updated: 2006-10-24

This page contains an extensive list of Mozilla preferences, and is a HTML conversion of the information contained in the Preferential extension. While it is not complete, it is one of the most comprehensive resources available on the Internet. Please e-mail GuruJ@mbox.com.au with any new preference descriptions or errata. This list applies to Mozilla Suite, Mozilla Firebird, Mozilla Thunderbird, other derivatives and all related extensions!

NB: All references to FOO or BAR are wildcards. Replace this with the number or text of your choice here (exactly which depends on the context) and Mozilla should pick up all preferences that match this pattern.

autocomplete
Autocomplete preferences
autocomplete.grab_during_popup
(X11 only) Specify if the autocomplete pop-up control should grab the keyboard. This option can be set to false to work around the bug where XKB does not set mode switching properly.
The default value is true.
Setting this preference dynamically takes effect in the next autocomplete session.

See also: autocomplete.ungrab_during_mode_switch
autocomplete.ungrab_during_mode_switch
(X11 only) This option complements autocomplete.grab_during_popup. If both preferences are true, then during an autocomplete session the application will release/grab keyboard when a mode-switch key is pressed/released.
If either of these two preferences is false, then the application will not release keyboard grab during an autocomplete session.
The default value is true.
Setting this preference dynamically takes effect in the next autocomplete session.

See also: autocomplete.grab_during_popup

network.cookie.p3p
Look at the p3p field in the server's response header to determine the privacy policy, and then processes the cookie accordingly. 'f' means 'flag', 'd' means 'downgrade to session', 'a' means 'accept', and 'r' means 'reject'. The letters in the 1st, 3rd, 5th, and 7th position are for first parties, and those in the even-numbered positions are for third parties. The four situations are: no policy, collects personal info without permission, collects personal info only with permission, and collects no personal info.
